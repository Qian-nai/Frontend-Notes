响应式布局实战

响应式网页设计（Responsive Web Design）是一种网页设计方法， 使网页能够自动适应不同设备的屏幕尺寸和分辨率。

移动端优先

移动优先（Mobile First）是一种设计策略，先为移动设备设计， 然后通过媒体查询逐步增强大屏幕的体验。

断点选择

常见的断点包括：768px（手机/平板分界）、1024px（平板/桌面分界）。 根据项目需求，可以添加更多断点。

弹性布局

使用 Flexbox 或 Grid 布局可以更轻松地实现响应式设计。 配合媒体查询，可以精确控制不同屏幕下的布局表现。

实现要点解析

1. 移动端优先策略

基础样式针对移动端编写，无需媒体查询。容器宽度100%，侧边栏和主内容自然上下排列。

2. 手机端断点（max-width: 767px）

@media screen and (max-width: 767px) {
    .sidebar {
        text-align: center;  /* 侧边栏文字居中 */
    }
}

3. 平板端断点（768px - 1024px）

@media screen and (min-width: 768px) and (max-width: 1024px) {
    .container {
        display: flex;       /* 启用 flex 布局 */
        flex-wrap: nowrap;   /* 不换行 */
    }
    .sidebar {
        width: 280px;        /* 左侧固定宽度 */
        flex-shrink: 0;      /* 禁止收缩 */
    }
    .main-content {
        flex: 1;             /* 右侧自适应填充 */
    }
}

4. 桌面端断点（min-width: 1025px）

@media screen and (min-width: 1025px) {
    .container {
        display: flex;
        max-width: 1200px;   /* 最大宽度限制 */
        margin: 0 auto;      /* 水平居中 */
    }
    .sidebar {
        width: 25%;          /* 左侧百分比宽度 */
    }
    .main-content {
        width: 75%;          /* 右侧百分比宽度 */
    }
}

5. 关键技巧

- 使用 flex-shrink: 0 防止固定宽度的侧边栏被压缩
- 使用 flex: 1 让主内容区自适应填充剩余空间
- 桌面端使用 max-width + margin: 0 auto 实现居中
- 断点选择 768px 和 1024px 是业界常用标准
